# 仁爱社团官网产品文档

# 产品定位
**仁爱社团官网是南阳理工学院计算机与软件学院仁爱社团的主要且唯一官网。主要介绍社团发展历程，社团各方向介绍区及交流讨论区。并附带完整博客一条龙服务，OJ,ACM友情链接服务。**
# 设计目标

127.0.0.1
1. **外观高效整洁，层次分明**
1. **高性能，高可靠，高安全，低耦合，高内聚**
1. **在网络通畅的状态下实现零等待**
1. **可以保存历届成员上传的数据及博客数据**
# **技术栈和系统要求**
**前端：围绕JavaScript技术栈构建，使用ES6标准  核心框架 React，Vue    **
**后端：围绕Go，MySQL，Redis，MongoDB构建技术栈**
**运维：docker-compose   docker**
**其他：Git，RabbitMQ，Grpc**
# 一  前台功能
## 1. 登陆注册
**前台无需注册也可进行访问查看，但会受到颇多限制，会在下面的功能处做详细说明。前台名义上只允许仁爱社团成员进行注册登录，且注册后需要后台审核通过后才可进行登录。注册只允许邮箱加用户名绑定的形式进行注册（其他注册方式暂未开放......）。登录采用用户名或邮箱双重登录的形式，暂不支持第三方登录（QQ，微信，钉钉等），暂不支持扫码登录**
### 1.1    邮箱-用户名注册
**前台注册为邮箱加用户名绑定的形式进行注册，注册需填写用户名，昵称，密码，已申请邮箱。点击“发送验证码”按钮，对填写邮箱发送随机6位验证码。注册用户填写完整数据后，方可进行注册。**
**注：**

- **邮箱不建议使用谷歌，微软邮箱，一个邮箱只能注册一次**
- **密码至少8位，最多20位，需输入两次**
- **所有数据不可为空**
- **验证码发送间隔为60s**
- **昵称允许重复**
- **用户名最少4个字符，最多20个字符，不允许重复，只可以是字母加数字的组成形式，且只可字母开头，不允许使用符号**
#### 1.1.1    注册审核
**注册成功后需经过后台审核通过才算注册成功，如果审核通过，注册邮箱会收到审核通过的通知，如失败则会收到注册失败的通知**
**
### 1.2    登录
**用户可以点击登录进行登录操作，需填写用户名或者注册邮箱，密码和随机生成的验证码。登陆成功后显示用户头像和昵称**
#### 1.2.1    免密登录
**用户登录成功后打开社团网站默认为登录状态，免密为15天，15天后需再次登录**
#### 1.2.2    退出登录
**用户在登陆状态下可以点击退出按钮退出登录，退出登陆后下次进入网站需重新登陆**
#### 1.2.3    单点登录
**一个账号只允许登录一个浏览器，不可同时登录多个浏览器（不同电脑的同一种浏览器，一台电脑的不同浏览器都算多个浏览器）**
**
### 1.3    忘记密码
**登录窗口提供忘记密码功能，需要用户填写注册邮箱发送邮箱验证码，验证成功后方可进行修改密码操作。修改密码需填写两次，修改成功后直接默认新密码登录。**

# 二  后台功能
## 1. 登陆注册
**后台管理系统需先注册，登录成功后才可进入管理系统。其目的是防止无关人员恶意注册对社团网站造成不可挽回的损失。后台管理系统暂只支持邮箱注册，不支持手机注册，不支持第三方登录，名义上后台管理只能注册一次（社团祖传邮箱注册）。**
### 1.1    邮箱注册
**邮箱注册，用户需填写已注册邮箱，特定注册码。点击“发送验证码”，并填写对应正确的邮箱验证码及特定注册码后方可进行注册**
**
**注：**

- **邮箱建议不要使用谷歌，微软邮箱，因为有可能会被当作垃圾邮件而收不到**
- **发送验证码的时间间隔为60s**
- **邮箱只允许注册一次**
- **所有数据必须填写完整**
### 1.2    登录
**输入注册邮箱，点击获取验证码并填写随机验证码方可进行登陆操作**
#### 1.2.1    免密登录
**后台管理不做免密登录**
#### 1.2.2    退出登录
**用户点击退出按钮，退出后台管理系统**
#### 1.2.3    单点登录
**一个账号只允许登录一个浏览器，不可同时登录多个浏览器（不同电脑的同一种浏览器，一台电脑的不同浏览器都算多个浏览器）**
**
## 2. 成员管理
### 2.1    审核前台用户注册
**前台用户注册后需要审核才可登录，后台审核时需显示前台注册时的用户名，昵称，邮箱，注册时间。后台可以通过或驳回前台用户的注册申请，通过或驳回后会发送对应类型的邮箱信息通知用户是否注册通过**

- **可根据注册时间升降序**
- **可根据邮箱模糊查询注册申请**
- **可根据用户名模糊查询注册申请**
- **可根据昵称模糊查询注册申请**
### 2.2    审核用户邮箱变更申请
**后台管理审核前台用户变更绑定邮箱的申请。需显示申请人的个人信息，如用户名，头像，昵称，性别，原注册邮箱，新邮箱，申请变更理由。后台管理可以通过该邮箱变更申请，通过后，前台用户“新邮箱”会收到邮箱变更成功的通知。驳回邮箱需填写驳回理由，驳回后，前台用户“新邮箱”会收到驳回通知并显示驳回理由。**
### 2.3    注册人员信息管理
**显示所有已注册的人员基本信息，如头像，用户名，昵称，性别，学习方向，注册邮箱，QQ号，注册时间，社团职位，后台管理可修改前台注册人员的个人信息。**

- **注册时间不需要修改**
- **用户名名义上不可修改**
- **社团职位由后台管理设置，目前分为社长，副社长，社员**
## 3. 社团信息管理
### 3.1    编写社团简介
**后台管理员可以查看已经编写或未编写的社团简介。第一次编写完成后，点击“发布”按钮即可发布编写的社团简介，前台则会对应更新为最新的社团简介。后继修改后，点击更新按钮即可更社团简介。**
